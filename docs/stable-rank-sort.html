Stable rank sort from shared read-only memory

The idea is bubble-sort-like in spirit: each value discovers how far it should rise or sink. Instead of adjacent swaps, it counts the values below it and jumps directly to its final stable position.

Core Rule

For source element input[i], every worker can read the whole input array and compute one final destination index.

less = count(input[j] < input[i]) equal_before = count(input[j] == input[i] and j < i) final_index = less + equal_before

Stability

The equal_before term preserves duplicate order. If three values compare equal, the one with the earlier source index receives the earlier sorted index.

In Rust code, equality should mean cmp() == Equal, not necessarily full structural equality, so keyed payloads stay stable.

Worker Shape

With 100 values and 10 workers, each worker can own 10 source indexes. Each worker streams over the same read-only input and updates counters for its owned candidates.

0..10

Careful Parts

  • Input must be read-only while workers are ranking values.
  • Each source index must be handled by exactly one worker.
  • Output writes are unique because stable ranks are unique.
  • Total work is O(n squared), even when many cores share it.
  • Speedup depends on cache behavior, memory bandwidth, and core count.

No-alloc Rust Shape

for i in worker_range {
    let mut rank = 0;

    for j in 0..input.len() {
        match input[j].cmp(&input[i]) {
            Ordering::Less => rank += 1,
            Ordering::Equal if j < i => rank += 1,
            _ => {}
        }
    }

    output[rank] = input[i];
}
